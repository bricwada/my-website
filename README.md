## サイト概要

New Emolgame は、ポケモンバトル（主にダブルバトル / ランクマッチ / 公式大会）に関する情報をまとめて届けることを目的としたメディアサイトです。

主なコンテンツは以下の通りです。

- 上位プレイヤーが使用した **構築記事** の紹介
- 現行 **レギュレーション** の早見表とルール解説
- **大会情報** とスケジュール
- これからポケモンバトルを始める人向けの **初心者ガイド**

エモンガをマスコットに据え、空色 × 黄色を基調とした明るいトンマナで構成しています。

## ページ構成

| ファイル          | ページ名         | 概要                                             |
| ----------------- | ---------------- | ------------------------------------------------ |
| `index.html`      | トップ           | サイトのトップページ。新着記事・特集をまとめる   |
| `builds.html`     | 構築紹介         | 構築記事の一覧。人気記事ランキング・タグ絞り込み |
| `regulation.html` | レギュレーション | 現行レギュレーションの早見表とルール詳細         |
| `tournament.html` | 大会情報         | 直近の大会スケジュールと関連リンク               |
| `beginner.html`   | 初心者向け       | はじめての人向けのステップ解説と Q&A             |

各ページのヘッダー・フッター・カラーパレットは共通スタイル（`style.css` および各ページ内 `<style>`）でそろえています。

## 技術スタック

- HTML5 / CSS3（バニラ）
- Google Fonts
  - Noto Sans JP
  - M PLUS Rounded 1c
  - Bebas Neue
- ホスティング: GitHub Pages
- カスタムドメイン: `new-emolgame.com`（`CNAME` ファイルで設定）

ビルドツールやフロントエンドフレームワーク（React / Vue など）は使用していません。HTML / CSS のみで完結する純粋な静的サイトです。

## ローカルでの動作確認

静的ファイルだけで動くため、ブラウザで直接 HTML ファイルを開いても大きな問題はありませんが、相対パスや Fetch などの挙動を本番に近づけるためにローカルサーバーを立てるのを推奨します。

このリポジトリではローカル開発に以下のツールを使っています。

- [`uv`](https://docs.astral.sh/uv/): Python 環境の管理（ローカルサーバーの起動に使用）
- [`just`](https://github.com/casey/just): タスクランナー（`make` の現代版）

### 1. uv のインストール（初回のみ）

```bash
brew install uv
```

`brew` 以外の導入方法は [uv のインストールガイド](https://docs.astral.sh/uv/getting-started/installation/) を参照してください。インストール後はバージョン確認しておくと安心です。

```bash
uv --version
```

### 2. just のインストール（初回のみ）

```bash
brew install just
```

`brew` 以外の導入方法は [just のインストールガイド](https://just.systems/man/en/chapter_4.html) を参照してください。

### 3. ローカルサーバーを起動

```bash
just up
```

ブラウザで http://localhost:8000/ を開いてください。ポートを変えたい場合は `just up 3000` のように引数で指定できます。

内部的には `uv run python -m http.server` を実行しており、Python は uv が自動で用意するのでシステム側に Python を入れておく必要はありません。

### 利用可能なタスク

レシピの一覧は次のコマンドで確認できます。

```bash
just
```

| コマンド             | 内容                                                        |
| -------------------- | ----------------------------------------------------------- |
| `just up [port]`     | ローカルサーバーを起動（デフォルト: 8000）                  |
| `just lint`          | HTML / CSS / フォーマットの lint をまとめて実行             |
| `just fmt`           | Prettier と Stylelint で自動修正                            |
| `just install-hooks` | git の pre-commit フックを有効化（初回のみ）                |
| `just pre-commit`    | 全ファイルに対して pre-commit を実行（CI と同等のチェック） |

### コミット時の自動チェック（pre-commit）

このリポジトリでは [`pre-commit`](https://pre-commit.com/) を使い、`git commit` のタイミングで Prettier / Stylelint / HTMLHint を自動実行する設定を入れています。
最初に一度だけフックを有効化してください（`uvx` 経由で実行されるので追加インストールは不要）。

```bash
just install-hooks
```

これ以降は `git commit` するたびに変更ファイルに対して lint・整形が走ります。整形によりファイルが書き換わった場合、コミットは中断されるので `git add` し直してから再コミットしてください。

任意のタイミングで全ファイルに対して走らせたい場合は次のコマンドが使えます。

```bash
just pre-commit
```

### just を使わない場合

just を経由せずに直接サーバーを立てたい場合は、uv で同じコマンドを叩けば OK です。

```bash
uv run python -m http.server 8000
```

VS Code を使っている場合は `Live Server` 拡張機能を使うとファイル保存時の自動リロードが便利です。

## デプロイ

このリポジトリは GitHub Pages により自動公開されています。

1. `main` ブランチに変更を push する
2. GitHub Pages がリポジトリ直下のファイルを公開
3. `CNAME` に記載されたドメイン（`new-emolgame.com`）でアクセス可能

ドメインの DNS 設定は GitHub Pages のカスタムドメイン要件に従ってください。

> 注意: `CNAME` ファイルは GitHub Pages のカスタムドメイン用設定です。誤って削除・変更するとサイトのドメインが切り替わってしまうため、変更時は必ず確認してください。

## 開発ルール

- 文字コード: UTF-8
- 言語属性: `<html lang="ja">`
- インデント: スペース 2
- カラーパレットは `:root` の CSS 変数（`--sky`, `--yellow`, `--dark` など）を再利用する
- 新しいページを追加するときは
  - 既存ページのヘッダー / ナビ / フッターを揃える
  - `<title>` を `ページ名 | New Emolgame` 形式にそろえる
  - スマホ表示（375px〜）でレイアウト崩れがないことを確認する
